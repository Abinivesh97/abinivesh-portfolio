Layers 6 & 5: Presentation & Session

The “translator” and the “conversation manager”

OSI Layers 6 & 5

L6

Presentation

Translates formats and can encrypt

Analogy

Like translating your letter into a universal language, then writing it in “invisible ink” so only the right person can read it.

Makes data readable everywhere

Example: text encoding (UTF‑8) so letters and emojis look the same on different devices.

Can compress or format

Examples: JPEG/PNG images, MP3 audio, video formats.

Can encrypt for privacy

Common example: TLS (the “S” in HTTPS) helps keep web browsing private.

Devices that often work here

Load balancer (SSL/TLS)

Can handle encryption for many servers.

TLS offloader

Specialized device for decrypt/encrypt.

L5

Session

Starts, manages, and ends conversations

Analogy

Like calling the post office and saying: “I’m about to send a bunch of letters—keep this line open until I’m done.”

Opens a session

“We’re connected; let’s start talking.”

Keeps the conversation organized

Helps apps keep track of “who is still logged in” or “which call is active.”

Closes the session

“We’re done—clean up and free resources.”

Devices you might hear about

Session Border Controller

Manages VoIP call sessions.

VPN concentrator

Manages secure VPN “sessions.”

Beginner memory trick

Presentation = “How should the message look & stay secret?” • Session = “Are we still in the same conversation?”
