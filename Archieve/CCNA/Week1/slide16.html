Seeing the Layers in Real Life: Wireshark ↔ OSI

The Packet Details pane is basically a “layer-by-layer label list” for one captured packet

Goal: connect names you see → OSI layers

What you’re looking at

Wireshark breaks packets into familiar layers

Expand arrows in Packet Details

You’ll see items like Ethernet II, IP, and TCP/UDP.

Think: “Which label is this?”

MAC (local) • IP (global) • Ports (which app)

HTTPS note

You may see TLS instead of HTTP because the website is encrypted.

Beginner reading checklist

1) Find the app protocol

DNS, HTTP, TLS (answers “what is happening?”)

2) Check IP addresses

Who is talking to who on the internet

3) Check ports

Which app/service on each device

4) Check MAC addresses

How the packet moved locally

Map Wireshark fields to the OSI model
