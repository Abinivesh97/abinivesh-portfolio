Wireshark: The X‑Ray Machine for Networks

A free tool that lets you see real packets moving through your network

Section 4: Wireshark Demo

Plain-English definition

Wireshark takes snapshots of network “envelopes.”

It captures packets.

Each packet contains headers (labels) like MAC, IP, and TCP/UDP.

It shows layers like a “peelable onion.”

You can expand a packet to see: Ethernet → IP → TCP/UDP.

It shows timing and performance clues.

Example: “How long did that request take?” or “Are packets lost?”

What beginners can “read”

Without knowing coding or math

Who talks to who

IP addresses + ports.

Which protocol

DNS, HTTP, TLS, TCP, UDP.

Errors & retries

Drops, retransmissions, resets.

Speed clues

Delays, RTT, slow DNS.

With HTTPS, Wireshark shows the “outside labels” (IP/TCP), but the web contents are encrypted.

Ethics & safety (non‑negotiable)

Capture only on networks you own or have permission to analyze.

Don’t capture other people’s private traffic.
